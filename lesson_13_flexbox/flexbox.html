Basic concepts of flexbox

The flexible box layout module, also referred to as flexbox, is a one-dimensional layout model for distributing space between items and includes numerous alignment capabilities. Flexbox deals with layout in one dimension at a time, either as a row or as a column. This can be contrasted with the two-dimensional model of grid layout, which controls columns and rows together. Even when flex items are allowed to wrap, each line behaves as if it were a flex container of its own.

The two axes of flexbox

Flexbox has two axes, the main axis and the cross axis. The main axis is defined by the flex-direction property, and the cross axis runs perpendicular to it.

The main axis

The main axis is defined by flex-direction, which has four possible values:

  • row
  • row-reverse
  • column
  • column-reverse

The cross axis

The cross axis runs perpendicular to the main axis. If your flex-direction (main axis) is set to row or row-reverse the cross axis runs down the columns. If your main axis is column or column-reverse then the cross axis runs along the rows.

Start and end lines

Flexbox makes no asumption about the writing mode of the document. Flexbox doesn't assume all lines of text start at the top left of a document and run towers the right-hand side.

For example, if the flex-direction and I am working in English, then the start edge of the main axis will be on the left, and the end edge on the rigth. If I were to work in Arabic, the start edge of my main axis would be on the right and the end edge on the left.

The flex container

The flex container is an area of a document that is laid out using flexbox. To create a flex container, set the area's display property to flex. This causes the direct children of that container to become flex items.

Initial Values

  • Items display in a row(the flex-direction property's default value is row
  • Items start from the start edge of the main axis
  • The items do not stretch on the main dimension but can shrink (a flex-item's flex-grow property's default value is 0 and its flex-shrink property's default value is 1)
  • The items will stretch to fill the size of the cross-axis (the align-items property's default value is stretch)
  • The flex-item's flex-basis property's default value is auto. This means that it will be equal to the flex item width and height
  • All the items will be in a single row (the flex-wrap property's value is nowrap), overflowing their container if their combined width/height exceeds the containin element width/heigth

    • In summary, your items will all line up in a row, using the size of the content as their size in the main axis. If there are more items that can fit in the container, they will not wrap but will instead overflow.

Multi-line flex containers with flex-wrap

Although flexbox is a one dimensional model, it is possible to make flex items wrap across multiple lines. If you do this, you should consider each line as a new felx container. Any space distribution will happen across each line, without reference to the previous or subsequent lines.

To cause wrapping behavior add the property flex-wrap: wrap.

The flex-flow shorthand

You can combine the two properties flex-direction and flex-wrap into the flex-flow shorthand

flex-flow: <flex-direction> <flex-wrap>

Properties applied to flex item

  • flex-grow
  • flex-shrink
  • flex-basis

The flex-basis property

Defines the size of that item in terms of the space it leaves as available space. The initial value of this property is auto -- in this case the browser looks to see if the item has a size. If the items have a width of 100pixels, then that size is used as the flex-basis.

If the item doesn't have a size then the content's size is used as the flex-basis.

The flex-grow property

When the flex-grow property is set to a positive integer, if there is available space, the flex item can grow along the main axis from its flex-basis. Whether the item stretches to take up all the available space on the axis, or just a portion of the space depends on if the other items are allowed to grow too and the value of their flex-grow properties

For example, if we gave all of our items a flex-grow: 1 then the available space in the flex container would be equally shared between our items and they would stretch to fill the container on the main axis.

On the other hand, if we have one of our items flex-grow: 2, and the rest a value of 1, then the item with a value of 2 will grow twice as fast as the other items.
