Basic concepts of flexbox

The flexible box layout module, also referred to as flexbox, is a one-dimensional layout model for distributing space between items and includes numerous alignment capabilities. Flexbox deals with layout in one dimension at a time, either as a row or as a column. This can be contrasted with the two-dimensional model of grid layout, which controls columns and rows together. Even when flex items are allowed to wrap, each line behaves as if it were a flex container of its own.

The two axes of flexbox

Flexbox has two axes, the main axis and the cross axis. The main axis is defined by the flex-direction property, and the cross axis runs perpendicular to it.

The main axis

The main axis is defined by flex-direction, which has four possible values:

  • row
  • row-reverse
  • column
  • column-reverse

The cross axis

The cross axis runs perpendicular to the main axis. If your flex-direction (main axis) is set to row or row-reverse the cross axis runs down the columns. If your main axis is column or column-reverse then the cross axis runs along the rows.

Start and end lines

Flexbox makes no asumption about the writing mode of the document. Flexbox doesn't assume all lines of text start at the top left of a document and run towers the right-hand side.

For example, if the flex-direction and I am working in English, then the start edge of the main axis will be on the left, and the end edge on the rigth. If I were to work in Arabic, the start edge of my main axis would be on the right and the end edge on the left.

The flex container

The flex container is an area of a document that is laid out using flexbox. To create a flex container, set the area's display property to flex. This causes the direct children of that container to become flex items.

Initial Values

  • Items display in a row(the flex-direction property's default value is row
  • Items start from the start edge of the main axis
  • The items do not stretch on the main dimension but can shrink (a flex-item's flex-grow property's default value is 0 and its flex-shrink property's default value is 1)
  • The items will stretch to fill the size of the cross-axis (the align-items property's default value is stretch)
  • The flex-item's flex-basis property's default value is auto. This means that it will be equal to the flex item width and height
  • All the items will be in a single row (the flex-wrap property's value is nowrap), overflowing their container if their combined width/height exceeds the containin element width/heigth

    • In summary, your items will all line up in a row, using the size of the content as their size in the main axis. If there are more items that can fit in the container, they will not wrap but will instead overflow.

Multi-line flex containers with flex-wrap

Although flexbox is a one dimensional model, it is possible to make flex items wrap across multiple lines. If you do this, you should consider each line as a new felx container. Any space distribution will happen across each line, without reference to the previous or subsequent lines.

To cause wrapping behavior add the property flex-wrap: wrap.

The flex-flow shorthand

You can combine the two properties flex-direction and flex-wrap into the flex-flow shorthand

flex-flow: <flex-direction> <flex-wrap>

Properties applied to flex item

  • flex-grow
  • flex-shrink
  • flex-basis

The flex-basis property

Defines the size of that item in terms of the space it leaves as available space. The initial value of this property is auto -- in this case the browser looks to see if the item has a size. If the items have a width of 100pixels, then that size is used as the flex-basis.

If the item doesn't have a size then the content's size is used as the flex-basis.

The flex-grow property

When the flex-grow property is set to a positive integer, if there is available space, the flex item can grow along the main axis from its flex-basis. Whether the item stretches to take up all the available space on the axis, or just a portion of the space depends on if the other items are allowed to grow too and the value of their flex-grow properties

For example, if we gave all of our items a flex-grow: 1 then the available space in the flex container would be equally shared between our items and they would stretch to fill the container on the main axis.

On the other hand, if we have one of our items flex-grow: 2, and the rest a value of 1, then the item with a value of 2 will grow twice as fast as the other items.

The flex-shrink property

flex-shrink controls how the elements become smaller. If we do not have enough space in the container to lay out our items, and flex-shrink is set to a positive integer, then the item can become smaller than the flex-basis. Different items can have different values which will cause some items to shrink faster than others. An item can shrink down to its min-content size.

The flex shorthand

The flex shorthand allows you to combine flex-grow, flex-shrink, and flex-basis into one in the following order

flex: <flex-grow> <flex-shrink> <flex-basis>

Alignment, justification and distribution of free space between items

align-items

This property aligns all the flex items on the cross axis. The initial value for this property is stretch which is why flex items stretch to the h eight of the flex container by default. The following are the possible values for align-items

  • stretch
  • flex-start
  • flex-end
  • start
  • end
  • center
  • baseline
  • last baseline

justify-content

This property is used to align the items on the main axis, the direction in which flex-direction has set the flow. The following are the possible values for justify-content

  • start
  • end
  • left
  • right
  • normal
  • flex-start
  • flex-end
  • center
  • space-around
  • space-between
  • space-evenly
  • stretch

justify-items

The justify-items property is ignored in flexbox layouts.

place-items and place-content

place-items: <align-items> <justify-items>

place-content: <align-content> <justify-content>

The align-content property only affects flex containers that wrap.

Relationship of flexbox to other layout methods

Flexbox and grid layout

Flexbox is a one dimensional layout method whereas grid layout is a two dimensional layout method. Flex items are allowed to wrap but, once they do, each line behaves as if it were a flex container of its own. When space is distributed, flexbox does not look at the placemenet of items in other rows and tries to line things up with each other. In grid we can control the layout in both rows and columns

In grid layout, you do the majority of sizing specification on the container, setting up tracks and then placing items into them. In flexbox, while you create a flex container and set the direction at that level, any control over item sizing needs to happen on the items themselves!

As a general rule, if you are setting widths on felx items to make items in one row of a wrapped flex container line up with items above them, you should instead opt for a two-dimensional grid layout!

Flexbox and other layout methods

If an item was floated or cleared and the becomes flex item due to the parent having display: flex applied, the floating and clearing will no longer happen, and the item will not be taking out of normal flow in the way that floats are.

If you have used the vertical-align property, as used with inline-block or table layout alignments, this will no longer affect the item and you can use the alignment properties of flexbox instead.

Flexbox and display: contents

display: contents is described as follows

"The element itself does not generate any boxes, but its children and pseudo-elements still generate boxes as normal. For the purposes of box generation and layout, the element must be treated as if it has been replaced with its children and pseudo-elements in the document tree"

Image we have a clox container with three flex items. One has two elements nested inside, which would not participate in flex layout because flex layout only applies to direct children of a flex container. When we add display: contents to the one element that is a wrapper around nested elements, the wrapper seems to disappear from the layout, allowing the two sub-children to be laid out as if they were direct children of the flex container.

Note that this only removes the box from the layout; the sub-children don't become direct children of the flex container.

Aligning items in a flex container

Using alignment in flexbox

Flexbox provides several properties to control alignment and spacing, with align-items and justify-content being fundamental for centering elements.

Properties for controlling alignment in flexbox

  • justify-content: Controls the alignment of all items on the main axis
  • align-items: Controls the alignment of all items on the cross axis.
  • align-self: Controls the alignment of an individual flex item on the cross axis.
  • align-content: Controls the sapce between flex lines on the cross axis
  • gap, column-gap, and row-gap: Used to create gaps or gutters between flex items.

Aligning items on the cross axis

The align-items property, set on the flex container, and the align-sefl property, set on the flex items, control the alignment of lex items on the cross axis. The cross axis runs down the columns if flex-direction: row and along the rows if flex-direction: column.

By default, the items will stretch to match the height of the tallest item, as that item defines the height of the items on the cross axis. If the flex container has a height set, the items will stretch to that height, regardless of how much content is in each item.

Aligning one item with align-self

The align-items property sets the align-self property on all of the flex items as a group. You can explicitly delcare the align-self property to target a single item.

Aligning content on the cross axis with the align-content property

When the flex items are allowed to wrap across multiple lines, the align-content property can be used to control the distribution of space between the lines.

Aligning content on the main axis

Now that we have seen how alingment works on the cross axis with align-items and align-content, we can take a look at the main axis. Here we only have one property available -- justify-content. This is because we are only dealing with items as a group on the main axis. With justify-content we control what happens with available space, should there be more space than is needed to display the items.

Alignment and writing modes

With all these alignment methods, the values of start and end are writing mode-aware. If the value of justify-content is start and the writing mode is left-to-right, as in English, the items will align starting at the left side of the container.

However if the writing mode is right-to-left as in Arabic, the items will line up starting at the right side of the container.

To change the writing mode, set the direction property value to rtl to force a right-to-left flow, or set to ltr for left-to-right flow.
