Grid layout using line-based placement

Positioning items by line number

If we do not place items on the grid in any way, they will lay out according to the auto-placement rules, one item in each cell. We can use line-based placement to control where items sit on the grid.


                    .box1 {
                        grid-column-start: 1;
                        grid-column-end: 2;
                        grid-row-start: 1;
                        grid-row-end: 4;
                      }

As you position some items, other items on the grid will continue to be laid out using auto placement rules.

The grid-column and grid-row shorthands

The grid-column-start and grid-column-end properties can be combined into grid-column

The grid-row-start and grid-row-end can be combined into grid-row


                    .box1 {
                        grid-column: 1 / 2;
                        grid-row: 1 / 4;
                      }
                      .box2 {
                        grid-column: 3 / 4;
                        grid-row: 1 / 3;
                      }
                      .box3 {
                        grid-column: 2 / 3;
                        grid-row: 1 / 2;
                      }
                      .box4 {
                        grid-column: 2 / 4;
                        grid-row: 3 / 4;
                      }

Default spans

If an item only spans on track, you can omit the grid-column-end or grid-row-end values.


                    .box1 {
                        grid-column-start: 1;
                        grid-row-start: 1;
                        grid-row-end: 4;
                      }

You can use default spans with grid-column and grid-row 


                    .box1 {
                        grid-column: 1;
                        grid-row: 1 / 4;
                      }

The grid-area property

Instead of using grid-column and grid-row, we can define each area with a grid-area property

The order of the values is as follows: grid-area: <grid-row-start> <grid-column-start> <grid-row-end> <grid-column-end>


                    .box1 {
                        grid-area: 1 / 1 / 4 / 2;
                      }

Gutters or Alleys

CSS grid includes the ability to adad gutterse between column and row tracks with the column-gap and row-gap properties. Gaps only appear between tracks of the grid, they do not add space to the top and bottom, left or right of the container. Anything starting at the line starts after the gap, and you cannot addres the gap or place anything into it.


                    .wrapper {
                        display: grid;
                        grid-template-columns: repeat(3, 1fr);
                        grid-template-rows: repeat(3, 100px);
                        column-gap: 20px;
                        row-gap: 1em;
                    }

The gap shorthand

The two gap properties can be expresed as a shorthand gap

gap: <row-gap> <column-gap>

gap: 1em 20px;

If you only give one value for gap it will apply to both column and row gaps.

Using the span keyword

In addition to specifying the start and end lines by number, you can specify the start line and then the number of tracks you would like the area to span using the span keyword.

The following examples create the same grid area. The area will start at line 1 and end 3 lines from line 1; that is, the area will end at line 4. 


                    .box1 {
                        grid-column: 1;
                        grid-row: 1 / span 3;
                    }

                    .box1 {
                        grid-column-start: 1;
                        grid-row-start: 1;
                        grid-row-end: span 3;
                    }

                    .box1 {
                        grid-column-start: 1;
                        grid-row-start: span 3;
                        grid-row-end: 4;
                    }
